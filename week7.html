Transitioning from continuous predictions to categorical outcomes using Odds, Logits, and Confusion Matrices.

01

The Classification Paradigm

Unlike regression, which predicts a continuous numerical value, Classification predicts a categorical variable (e.g., Success/Failure, True/False, 1/0). This is a Supervised Learning task because building the model requires both the input features ($X$) and the true historical labels ($Y$).

🛑 Why Standard Linear Regression Fails

If we try to directly model Probability ($p$) using $Y = \beta_0 + \beta_1X_1 \dots$, we face a mathematical barrier:

  • 1. Linear Output Range: $(-\infty, +\infty)$
  • 2. Probability Range: Strictly bounded to $[0, 1]$

Restricting a linear equation to never output values below 0 or above 1 requires incredibly complex constraints. Instead, we must mathematically transform the probability into a format that spans from $-\infty$ to $+\infty$.

02

Mathematical Transformations

To bridge the gap between probability and linear equations, we use a two-step mathematical translation: transforming Probability into Odds, and then Odds into Log-Odds (Logit).

Step 1: The Odds Ratio

The ratio of the probability of success to the probability of failure.

$$\text{Odds} = \frac{p}{1 - p}$$

Range: $[0, +\infty)$.
Issue: It resolves the lower bound but isn't symmetric. $p=0.5$ results in Odds $= 1$, which awkwardly splits the positive and negative classes.

Step 2: Log-Odds (Logit)

Taking the natural logarithm ($ln$) of the Odds ratio solves the symmetry and bounding issues entirely.

$$\text{Logit} = \ln\left(\frac{p}{1 - p}\right)$$

Range: $(-\infty, +\infty)$.
Perfection: Now $p=0.5$ maps exactly to $0$. Positive outputs dictate a positive class, negative outputs dictate a negative class.

∫

The Sigmoid (Logistic) Function

Because we modeled the Logit using the linear equation ($z = \beta_0 + \beta_1X_1 \dots$), we must mathematically invert it to extract the final predicted Probability ($p$).

$$p = \frac{1}{1 + e^{-z}}$$
03

Objective Function & Thresholding

Model Objective

In Linear Regression, the objective is to Minimize the Sum of Squared Errors (SSE).

In Logistic Regression, the objective completely changes. The model solver seeks to Maximize the Log-Likelihood of the odds.

Classification Thresholds

The Sigmoid function outputs a probability (e.g., $p = 0.7$). To finalize the classification, we compare it to a pre-decided Threshold (default is often $0.5$, but could be set to $0.8$ based on risk appetite).

  • If $p \ge \text{Threshold} \implies \text{Positive Class (1)}$
  • If $p < \text{Threshold} \implies \text{Negative Class (0)}$

05

Metrics & The Row/Column Tricks

Accuracy

Percentage of entirely correct predictions out of all samples processed.

$$\text{Accuracy} = \frac{TP + TN}{\text{Total Samples}}$$
The "Column" Trick

Precision

"Out of all the samples my model predicted as a specific class, how many were actually correct?"
Memory Trick: Use the COLUMN sum. The second letter in the denominator matches the class.

Precision (+ Class)

$$\frac{TP}{TP + F\textbf{P}}$$

Precision (- Class)

$$\frac{TN}{TN + F\textbf{N}}$$
The "Row" Trick

Recall

"Out of all the samples that actually belonged to a class, how many did the model successfully find?"
Memory Trick: Recall = Row sum. The second letter in the denominator is opposite of the class.

Recall (+ Class)

$$\frac{TP}{TP + F\textbf{N}}$$

Recall (- Class)

$$\frac{TN}{TN + F\textbf{P}}$$
